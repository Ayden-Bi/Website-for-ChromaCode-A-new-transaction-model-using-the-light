    ChromaCode:A new transaction model using the light

    Group Member：邓海伟，毕文东，孙雪涵，易兴睿


    This methods enabling hidden screen-camera channel. We hope to embed data frame into primary videos unobviously.
    So that when the video is being played, human eyes cannot percept the embedded data frame. Human eyes can only see the origin video content. But the data frame embedded in the video can be captured with a camera, and then decoded.
    This topic is a new transaction model using the light
    It trade communications between computer and mobiphones, later we will give a detailed description of this.


    The color Space and Structure of ChromaCode

    color Space


    We compare differences between non-uniform and uniform color spaces with their hue and lightness palettes. There are three annuluses in each color palette. The middle annulus has lightness L. The outer and inner annuluses have lightness L+ΔL and L-ΔL. For the palettes in non-uniform color space, color differences are not uniform.
    For example, color differences are more obvious here. While color differences are less obvious here.
    For palettes in uniform color space, The color differences are more perceptively uniform.
    Uniform color spaces suit human system better. Therefore we choose uniform color spaces for lightness modificvisionation.



    Archetecture of ChromaCode

    The main structure of Chromacode is as follows:

    • Origin data is encoded through concatenated [kɒn'kætɪneɪt] encoding
    • then modulated to generate data frame.
    • The data frame is then embedded into primary video with adaptive embedding, so that when the video is being played
    • Human eyes can only see origin video contents. But cameras can capture the embedded data frame
    • Then retrieve embedded data by demodulation and decoding.

    Demo



    There are mainly three key goals in hidden screen-camera communication. They are:

    • Invisibility: users cannot see the code
    • Reliability: it can tolerate noise
    • Throughput: it can achieve high volume of communications.

    However, these three key goals can be conflicting.
    For example, we can increase lightness adjusting amount to achieve more reliable data transmission, but more flickers may be observed by human eyes.
    Previous works usually made a trade-off among them. In our work, ChromaCode aims to achieve all three goals while retaining watching experience to users.
